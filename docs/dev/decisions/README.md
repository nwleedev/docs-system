# Development Decisions

This folder owns long-lived technical, architectural, documentation, or process decisions that affect `docs/dev` guidance.

## Include

- Accepted, deferred, superseded, rejected, partially superseded, or partially restored decisions.
- Decisions that promote future guidance to current rules.
- Decisions that change source-of-truth ownership, dependency strategy, verification strategy, or template structure.

## Exclude

- Ordinary code pattern teaching. Use `engineering/`.
- Current import or module rules without a decision trade-off. Use `architecture/`.
- Future proposals that are not accepted. Use `evolution/`.
- AI behavior rules without a durable engineering or documentation decision. Use `ai/`.

## Current Index

No detailed decision record is present yet. Add one only when a durable choice must remain discoverable after implementation.

## Evidence

Decision records should link the affected `docs/dev` folders and the repository evidence that made the decision necessary.
